云主机选型

发表于 2021-03-02

  1. 虚拟云主机：物理机上的虚拟机，一般配置为1核1GB，网页空间1GB，数据库200MB

    1. 用户侧：
      1. 最低价格：10-30元月
      2. 功能：个人网站,仅可以ftp上传，不开通ssh
      3. 难度：用户只需基础知识
      4. 客户端：FileZilla
    2. 供应商侧：
      1. 维护成本：一台物理机可以分成很多个网页空间，平分到单用户的成本低
      2. 技术难度：使用虚拟技术即可实现

  2. 轻量云应用（已经整合成3）：基于云Sass技术，可扩容。有应用镜像源。

    1. 用户侧

      1. 价格：100元/月

      2. 功能：可以架设各种预设服务。减轻用户的架势成本。ssh

      3. 难度：用户具有一般的服务器维护知识

      4. 客户端：终端工具

    2. 供应商侧：

      1. 维护成本：从计算资源池中分配计算性能，把机器性能压榨干净。成本低。
      2. 技术难度：需使用云化技术实现计算资源动态分配，对从业人员的技术要求高。技术成本高。

  3. 挂机宝：提供win环境的低配置虚拟机

    1. 用户侧

      1. 最低价格：3-10元/月

      2. 配置：常见配置为2核1GB内存11GB硬盘100MB共享带宽，共享IP，无显卡

      3. 特点：只能够远程连接桌面，不提供NAT服务，所以导致外网访问不了。不适合建站。

      4. 常见用途：QQ机器人，千牛等客服软件挂机服务，2D网页游戏挂机。

      5. 难度：掌握远程连接

      6. 稳定性：由于架设难度低，市场上存在个人卖家。很可能跑路，购买需谨慎。

      7. 客户端：mstsc，向日葵

    2. 供应商侧：

      1. 维护成本：低
      2. 技术难度：难度极低，只需要虚拟化提供远程端口即可

  4. 虚拟专属服务器（VPS）：相当于没有装php和mysql和网站应用的的虚拟机，国外的VPS共享带宽可达1Gb/s，所以有很多人买了做网关转发，可架设SSR。做成机场（翻墙软件）再卖给别人盈利。

    1. 用户侧：
      1. 最低价格：30元+/月
      2. 功能：作为服务器，其功能可以自定义
      3. 难度：用户具有专业的服务器维护知识
      4. 客户端：终端工具
    2. 供应商侧：
      1. 维护成本：一台物理机拆分成有限个VPS，平分到单用户的成本较高。
      2. 技术难度：使用虚拟技术即可实现

  5. 云服务器(ECS)：基于云Iass技术，可扩容。有应用镜像源（和2整合后）和常用系统镜像。

    1. 用户侧：

      1. 最低价格：按照配置计算，最低100元/月

      2. 功能：可以架设各种服务。开通ssh

      3. 难度：用户具有专业的服务器维护知识

      4. 客户端：终端工具

    2. 供应商侧：

      1. 维护成本：从计算资源池中分配计算性能，把机器性能压榨干净。成本低。
      2. 技术难度：需使用云化技术实现计算资源动态分配，对从业人员的技术要求高。技术成本高。

  6. 物理机：个人电脑远程租给客户

    1. 用户侧：

      1. 最低价格：100元+/月

      2. 功能：普通电脑可以干的事都可以干

      3. 常见用途：3D游戏挂机

      4. 难度：掌握远程连接

      5. 客户端：mstsc，向日葵

    2. 卖家侧：

      1. 特点：产品一般是网吧机器或者个人闲置计算机。配置参差不齐。

  7. 裸金属服务器：托管于机房的实体物理机

    1. 用户侧：
      1. 最低价格：1000+/月以上
      2. 功能：性能强悍，独占物理机资源
      3. 难度：用户具有专业的服务器维护知识
      4. 客户端：终端工具
    2. 供应商侧：
      1. 维护成本：一台物理机只能分配给一个用户。造成计算资源的浪费，成本高。
      2. 技术难度：只需提供远程控制技术。技术成本低。

结论

  1. 从技术的角度来说，以后会逐步淘汰虚拟技术，转向云技术+云存储技术。
  2. 配置选择
    1. 带宽因素：
      1. 经验：参考云主机如何选带宽和购买服务器时如何选择服务器带宽，在没CDN的情况下，2M足够在5秒以内打开纯网页，大并发访问的情况需要CDN，视频网站建站成本更高。同流量下，CDN的成本远低于升级虚拟机配置
      2. 策略：
        1. 多用户时，购买CDN服务
        2. 可以使用第三方云盘提供客户下载选项，减少成本
        3. 家庭100M带宽会有30M的下行带宽，适合建站
    2. 空间因素：
      1. 经验：参考网站空间服务器大小选择，不做大型网站，不做视频网站。1G足够用了。
  3. 结论：
    1. 博客：虚拟云主机，价格便宜，机器稳定，内容不易丢失
    2. 云端挂3D游戏：租用个人物理机，性能足，价格相对便宜
    3. 饥荒服务器：云服务器(ECS)、虚拟专属服务器（VPS），也可以选择购买第三方服务
    4. 在线玩游戏：云游戏平台，目前斗鱼云游戏是免费玩
    5. 电商/QQ机器人：挂机宝
    6. 企业数据：裸金属服务器、自建机房、云服务器(ECS)

小轿车驾驶总结

发表于 2021-03-02

熄火的原因

  1. 平地一档起步（易）
    1. 不踏脚刹时，离合器起得太快，阻力太大
    2. 踩脚刹时，此时虽然离合器起得不快，但是受脚刹影响
      1. 合器抬起时车身抖动，此时如果不放脚刹照样熄火
  2. 坡地一档起步（中等）
    1. 受车身重力影响，此时起步转速不足以带动受车身自重阻力影响下的车辆
      1. 此时，必须稍稍加油门同时缓慢抬起离合器，若不加油门则熄火
  3. 台阶一档起步（难）
    1. 此时车辆必须克服车重做功，抬升一定高度
      1. 此时，必须稍稍加油门同时缓慢抬起离合器，若不加油门则熄火
  4. 综上所述，熄火原因都是由于阻力太大引起的。

侧放停车

  1. 因为人坐在车内，视野受限。必须参照旁边物体，使得车身与参照物平行。
    1. 然后方向打死，即可倒车入库
      1. 在停车过程中，通过调整方向盘，使得车身与车库的墙壁（白线）平行。
        1. 继续倒车，直至车尾贴近车库（白线）尾端，停车。
  2. 难点:倒车时方向与方向盘的转向是相反的。
    1. 即，前进时，方向盘转的方向就是车头转向
      1. 后退时，方向盘转向是车尾转向。
    2. 如此考虑，倒车转S型曲线是最难的。

换挡

  1. 首先离合器踩到底，然后换挡，还完缓慢松开离合器

刹车

  1. 挺稳时使用手刹，完全抱死车轮。
  2. 刹车时，若遇到车身震动，可踏下离合器，较少阻力，避免熄火。

NET_ARG_baseFSM.h(25,25): error C3646: “init”: 未知重写说明符

简介

  • XML（Extensible Markup Language,可扩展标记语言）
  • XML 被设计用来传输和存储数据
  • 不是一种编程语言
  • 新增元素不会引起程序崩溃
  • 任意深度子元素

语言类比

  • XML和markdown和latex一样记录内容，但不是所见即所得
  • XML和json类似，都是数据交互格式
    • xml是重量级的，json是轻量级的
    • xml在传输过程中比较占带宽，json占带宽少，易于压缩
    • xml和json(JavaScript Object Notation, JS对象简谱) 都用在项目交互下，xml多用于做配置文件，json用于数据交互
    • json可用jackson，gson等方法解析，xml可用dom，sax，demo4j等方式解析

元素命名规则

  • 名称可以含字母、数字以及其他的字符
  • 名称不能以数字或者标点符号开始
  • 名称不能以字符 “xml”（或者 XML、Xml）开始
  • 名称不能包含空格

最佳命名习惯

  • 避免 “-“ 字符
  • 避免 “.” 字符
  • 避免 “:” 字符

属性和元素

  • 尽量使用子元素而非属性

    属性和子元素元素可以提供同样的信息

    • 属性无法包含多重的值（元素可以）
    • 属性无法描述树结构（元素可以）
    • 属性不易扩展（为未来的变化）
    • 属性难以阅读和维护

  • 元数据（有关数据的数据）应当存储为属性，而数据本身应当存储为元素

XML语法验证

  • 拥有正确语法的 XML 被称为“形式良好”的 XML
  • 通过 DTD 验证的 XML 是“合法”的 XML
    • DTD 的作用是定义 XML 文档的结构。它使用一系列合法的元素来定义文档结构。
    • 只要把 DOCTYPE 声明添加到代码中即可
