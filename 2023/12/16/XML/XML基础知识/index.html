XML 基础知识

[TOC]

简介

  • XML（Extensible Markup Language,可扩展标记语言）
  • XML 被设计用来传输和存储数据
  • 不是一种编程语言
  • 新增元素不会引起程序崩溃
  • 任意深度子元素

语言类比

  • XML 和 markdown 和 latex 一样记录内容，但不是所见即所得

  • XML 和 json 类似，都是数据交互格式
    - xml 是重量级的，json 是轻量级的

    • xml 在传输过程中比较占带宽，json 占带宽少，易于压缩
    • xml 和 json(JavaScript Object Notation, JS 对象简谱) 都用在项目交互下，xml 多用于做配置文件，json 用于数据交互
    • json 可用 jackson，gson 等方法解析，xml 可用 dom，sax，demo4j 等方式解析

语法

  1. 所有 XML 元素都须有关闭标签

  2. 若元素为空，可以折叠标签

  3. XML 标签对大小写敏感

  4. XML 必须正确地嵌套

  5. XML 文档必须有根元素

  6. XML 的属性值须加单/双引号

    &gt; > 大于
    &amp; & 和号
    &apos; ‘ 单引号
    &quot; “ 引号

  9. 在 XML 中，空格会被保留

  10. XML 以 LF 存储换行

  2. 本文和属性

元素命名规则

  • 名称可以含字母、数字以及其他的字符
  • 名称不能以数字或者标点符号开始
  • 名称不能以字符 “xml”（或者 XML、Xml）开始
  • 名称不能包含空格

最佳命名习惯

  • 避免 “-“ 字符
  • 避免 “.” 字符
  • 避免 “:” 字符

属性和元素

  • 尽量使用子元素而非属性

    属性和子元素元素可以提供同样的信息

    • 属性无法包含多重的值（元素可以）
    • 属性无法描述树结构（元素可以）
    • 属性不易扩展（为未来的变化）
    • 属性难以阅读和维护

  • 元数据（有关数据的数据）应当存储为属性，而数据本身应当存储为元素

XML 语法验证

  • 拥有正确语法的 XML 被称为“形式良好”的 XML
  • 通过 DTD 验证的 XML 是“合法”的 XML
    • DTD 的作用是定义 XML 文档的结构。它使用一系列合法的元素来定义文档结构。
    • 只要把 DOCTYPE 声明添加到代码中即可
